智能指针

1. shared_ptr

  1. 指向实际对象的指针 (T* ptr):

    • shared_ptr 内部有一个指针，指向通过 new 分配的实际对象。

  2. 引用计数 (shared_count):

    • 每个 shared_ptr 都有一个引用计数，记录有多少个 shared_ptr 共享对同一对象的所有权。

  3. 拷贝构造和赋值运算符的重载:

    • 在拷贝构造时，新的 shared_ptr 会指向相同的对象，并增加引用计数。
    • 在赋值运算符中，原有的 shared_ptr 会减少引用计数，如果引用计数变为零，表示没有其他 shared_ptr 指向该对象，此时释放对象的内存。然后将新的 shared_ptr 指向相同的对象，并增加引用计数。

  4. 析构函数的重载:

    • 在 shared_ptr 的析构函数中，引用计数减一。如果引用计数变为零，表示没有其他 shared_ptr 指向该对象，此时释放对象的内存。

  5. 线程安全问题:

    • 同一个 shared_ptr 被多个线程“读”是安全的：因为 shared_ptr 的引用计数是原子操作，多个线程对同一个 shared_ptr 进行读操作是安全的。引用计数的增加和减少操作都是原子的，不会导致竞态条件。

    • 同一个 shared_ptr 被多个线程“写”是不安全的：如果多个线程同时对同一个 shared_ptr 进行写操作，例如修改 shared_ptr 所指向的对象，这样的操作可能会导致竞态条件。因为这涉及到对实际对象的修改，而不仅仅是 shared_ptr 内部的引用计数。在这种情况下，需要使用其他同步机制来保证线程安全，比如使用互斥锁 (std::mutex)。

    • 共享引用计数的不同 shared_ptr 被多个线程“写”是安全的：如果多个线程操作不同的 shared_ptr 实例，但这些 shared_ptr 共享对同一对象的所有权（即引用同一块内存），这是安全的。因为引用计数是共享的，而且其操作是原子的。

类型推导

1. auto:

  1. 初始化： auto 可以让编译器在编译期推导变量的类型。使用时需要立即初始化，否则编译器无法推导类型。
  2. 一行多变量： 在一行定义多个变量时，各个变量的推导不能产生二义性，否则编译失败。
  3. 函数参数和非静态成员变量： 不能用作函数参数，在类中不能用作非静态成员变量。
  4. 数组和指针： 不能定义数组，但可以定义指针。
  5. 模板参数和引用类型： 无法推导出模板参数。在不声明为引用或指针时，会忽略等号右边的引用类型和cv限定；在声明为引用或指针时，会保留等号右边的引用和cv属性。

2. decltype:

  1. 表达式类型： 用于推导表达式的类型，实际上并不执行运算，而是在编译期进行分析。
  2. 类型保留： 不会像 auto 一样忽略引用和cv属性，而是保留表达式的引用和cv属性。
  3. 规则：
    • 若表达式是一个变量，decltype(exp) 和 exp 的类型相同。
    • 若表达式是一个函数调用，decltype(exp) 和函数返回值类型相同。
    • 对于其他情况，若表达式是左值，decltype(exp) 是表达式类型的左值引用。

右值引用：

左值和右值：

  1. 左值（Lvalue）：

    • 定义： 可以放在等号左边进行赋值操作的表达式。
    • 性质： 有名字，可以取地址，通常表示一个具体的对象或变量。

  2. 右值（Rvalue）：

    • 定义： 不能放在等号左边进行赋值操作的表达式。
    • 性质： 通常是临时的、无名的值，不可以取地址。

将亡值（Rvalue Reference）：

将亡值是指 C++11 中引入的右值引用的值。用 && 表示，它可以绑定到右值，并且在移动语义中发挥重要作用。

左值引用（Lvalue Reference）：

左值引用是对左值进行引用的类型，用 & 表示。左值引用可以修改被引用的对象，但不能绑定到右值。

右值引用（Rvalue Reference）：

右值引用是对右值进行引用的类型，用 && 表示。右值引用允许绑定到右值，并且在移动语义中用于资源的转移。

移动语义：

移动语义是 C++11 中引入的概念，通过右值引用实现。它允许在资源拥有者之间转移资源，避免不必要的深拷贝，提高性能。

深拷贝和浅拷贝：

  1. 浅拷贝： 多个对象共享同一块内存，修改一个对象的数据会影响其他对象。

  2. 深拷贝： 每个对象拥有自己的一份数据，修改一个对象的数据不会影响其他对象。

完美转发：

完美转发是 C++11 中引入的技术，用于在模板中保留参数的值类别。通过 std::forward 实现，在实现泛型函数时确保参数的类型信息不丢失。

std::forward 用于在泛型函数中将参数完美地转发到目标函数，保持参数的值类别。这在实现通用容器、智能指针等类时非常有用。

nullptr:

  1. 背景：

    • 在早期，C++使用 NULL 表示空指针，但不同编译器对 NULL 的定义可能有所不同，有的定义为 (void*)0，有的定义为 0。
    • 这可能导致在某些情况下发生类型混淆，例如在函数重载中。

  2. C++11 引入 nullptr:

    • nullptr 是 C++11 引入的关键字，用于代替 NULL。
    • nullptr 的类型是 nullptr_t，可以隐式转换为任何指针类型，避免了类型混淆问题。
    • 使用 nullptr 还能提供更好的类型安全性。

列表初始化:

  1. 背景：

    • C++提供了多种初始化方式，例如 int x = 0;，int x{0};，int x(0); 等。

  2. 列表初始化:

    • 使用花括号 {} 进行初始化称为列表初始化。
    • 对于内置类型，如果存在丢失信息的风险，使用列表初始化时，编译器会报错。
    • 列表初始化可以提高类型安全性，避免一些潜在的错误。

并发

在 C++ 中，为了进行线程同步和互斥，有一些常用的工具，其中包括 std::thread、std::lock_guard、std::unique_lock 等。以下是对这些工具的简要介绍：

1. std::thread

  • 默认构造函数： 创建一个空的 thread 执行对象。
  • 初始化构造函数： 创建一个 thread 对象，该对象可被 joinable，新产生的线程会调用 fn 函数，该函数的参数由 args 给出。
  • 拷贝构造函数 (被禁用)： 意味着 thread 不可被拷贝构造。
  • 移动构造函数： 调用成功之后 x 不代表任何 thread 执行对象。

注意：可被 joinable 的 thread 对象必须在它们销毁之前被主线程 join 或者将其设置为 detached。

2. std::lock_guard

std::lock_guard 是一个互斥量包装程序，提供了 RAII （Resource Acquisition Is Initialization）风格的机制，用于在作用域块的持续时间内拥有一个互斥量。创建 lock_guard 对象时，它将尝试获取提供给它的互斥锁的所有权。当控制流离开 lock_guard 对象的作用域时，lock_guard 析构并释放互斥量。

特点：

  1. 创建即加锁，作用域结束自动析构并解锁，无需手工解锁。
  2. 不能中途解锁，必须等作用域结束才解锁。
  3. 不能复制。

3. std::unique_lock

std::unique_lock 是一个通用的互斥量锁定包装器，具有比 std::lock_guard 更多的功能。它允许延迟锁定，限时深度锁定，递归锁定，锁定所有权的转移以及与条件变量一起使用。

特点：

  1. 创建时可以不锁定（通过指定第二个参数为 std::defer_lock），而在需要时再锁定。

  2. 可以随时加锁解锁。

  3. 作用域规则同 std::lock_guard，析构时自动释放锁。

  4. 不可复制，可移动。

  5. 条件变量需要该类型的锁作为参数（此时必须使用 std::unique_lock）。

    在这个示例中，std::unique_lock 对象 lock 创建时不立即锁定互斥量，然后在需要的时候手动调用 lock() 锁定互斥量，调用 unlock() 解锁互斥量。这种灵活性可以用于更复杂的锁定和解锁策略。
